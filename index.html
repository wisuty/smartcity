L1

Basic Chat — ถามตอบตรงๆ

เหมาะกับ: ค้นหาข้อมูล, อธิบาย concept, แปลเอกสาร

ใช้พิมพ์เหมือนคุยกับผู้เชี่ยวชาญ ไม่ต้องมี format พิเศษ Claude จะตอบตามบริบทที่ถาม

ง่ายสุด ไม่ต้อง setup ตอบทันที

L2

Structured Prompt — บอก Role + Goal + Format

เหมาะกับ: เขียนเอกสาร, สร้าง spec, เขียนโค้ดพื้นฐาน

เพิ่มประสิทธิภาพโดยบอก Claude ว่าคุณต้องการให้มันเป็นใคร, ทำอะไร, และ output จะออกมาแบบไหน

Role-based Output format Structured
💬 ตัวอย่าง Prompt L2
คุณคือ Network Architect ที่เชี่ยวชาญ WireGuard VPN งาน: ออกแบบ topology สำหรับ SmartCity ที่มี - 1 Master Node (Bangkok HQ) - 5 Client Nodes (กระจายตาม district) - IP Camera แต่ละ node 8 ตัว Output: อธิบายเป็น bullet point พร้อม subnet plan และ WireGuard config ตัวอย่าง
L3

Context-Rich — ให้ข้อมูลเต็ม วิเคราะห์ระบบซับซ้อน

เหมาะกับ: Architecture review, multi-system design, troubleshoot

วาง context ทั้งหมดที่มี เช่น spec hardware, ข้อจำกัด, สภาพแวดล้อม แล้วให้ Claude วิเคราะห์และแนะนำ

Full context Multi-step analysis System thinking
💬 ตัวอย่าง Prompt L3 (SmartCity)
Context ของระบบที่กำลังสร้าง: - Hardware: MikroTik RB5009 (Master), hEX S (Client nodes) - Camera: Hikvision 4MP H.265, 8 ตัว/node - VPN: WireGuard mesh, 5 nodes - Server: Ubuntu 22.04, Prometheus + Grafana + Frigate NVR - Network: Fiber ที่ HQ, 4G LTE สำรองที่ Client ปัญหา: Bandwidth ที่ node ชนบทมีแค่ 10Mbps กล้อง 8 ตัว H.265 1080p ดูไม่ออก ต้องการ: วิเคราะห์และออกแบบ sub-stream strategy + Frigate config ที่ optimize สำหรับ low-bandwidth พร้อม fallback plan ถ้า 4G ล่ม

HTML/React Python/Bash Word/Excel PDF

วิธีใช้ Claude สร้างระบบ SmartCity ทั้งหมด — 8 ขั้นตอน
🎯

Step 1 — กำหนด Requirements (L2)

บอก Claude ว่าระบบต้องการอะไร: จำนวน node, ประเภท hardware, งบประมาณ, bandwidth ที่มี แล้วให้ generate System Requirements Document

🏗️

Step 2 — ออกแบบ Network Architecture (L3)

วาง context เต็ม: MikroTik models, WireGuard topology, subnet plan, failover strategy ให้ Claude วิเคราะห์และแนะนำ architecture ที่เหมาะสม

⚙️

Step 3 — Generate Config Files (L4)

ให้ Claude สร้าง WireGuard config, MikroTik script, Prometheus config, Frigate config ที่พร้อม deploy จริง

📹

Step 4 — Camera Optimization (L3)

วิเคราะห์ bandwidth vs quality trade-off สำหรับ H.265 sub-stream กำหนด Frigate detection zones และ retention policy ตาม storage ที่มี

📊

Step 5 — สร้าง Command Center Dashboard (L4)

Generate HTML/React dashboard ที่แสดง node map, camera grid, bandwidth chart, alert system พร้อม dark theme

🤖

Step 6 — AI Monitoring Assistant (L5)

สร้าง App ที่มี Claude เป็น AI วิเคราะห์ alert อัตโนมัติ, แนะนำ troubleshoot step, generate incident report

👥

Step 7 — Peopleware: คู่มือและการฝึกอบรม (L2)

ให้ Claude เขียน User Manual ภาษาไทย, สร้าง SOP สำหรับ operator, ออกแบบ Training Program สำหรับทีม

💼

Step 8 — Sales Package (L3)

Generate ใบเสนอราคา, Proposal PDF, ROI Analysis, และ Presentation Deck สำหรับนำเสนอลูกค้า

🔧 Hardware Layer — Claude ช่วยได้
  • เปรียบเทียบ spec MikroTik รุ่นต่างๆ
  • วิเคราะห์ compatibility กล้อง + recorder
  • คำนวณ PoE budget และ power consumption
  • แนะนำ failover hardware
  • สร้าง BOM (Bill of Materials)
💻 Software Layer — Claude ช่วยได้
  • Generate WireGuard / OpenVPN config
  • เขียน Prometheus alerting rules
  • สร้าง Grafana dashboard JSON
  • Frigate NVR config optimization
  • เขียน Bash/Python automation scripts
👥 Peopleware — Claude ช่วยได้
  • เขียน Job Description สำหรับ operator
  • สร้าง Training Curriculum
  • ออกแบบ Escalation Process
  • เขียน SOP ภาษาไทย
  • สร้าง onboarding checklist
💰 Sales Layer — Claude ช่วยได้
  • เขียน Executive Summary
  • สร้าง Proposal Word/PDF
  • คำนวณ TCO และ ROI
  • เตรียม Q&A สำหรับลูกค้า
  • สร้าง Pitch Deck PowerPoint
⚡ Pro Tip: เริ่ม conversation ด้วยการให้ Claude "รู้จัก" ระบบของคุณก่อนเลย — วาง context ทั้งหมดครั้งเดียว แล้ว follow-up ต่อในสิ่งที่ต้องการ จะได้คำตอบที่ตรงกว่ามาก และสามารถต่อยอดได้ในทุก session
